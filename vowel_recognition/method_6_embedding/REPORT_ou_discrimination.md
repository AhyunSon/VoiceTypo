**작성일**: 2026-03-11
**프로젝트**: VoiceTypo Integrated
**대상 모듈**: vowel_recognition/method_6_embedding (Stage 2 이진 분류기)

---

## 1. 배경

### 1.1 시스템 구조

VoiceTypo의 모음 인식은 **2단계 분류기**로 동작한다:

- **Stage 1**: XLSR-53 Layer 16 임베딩 → SVM → 7개 모음 분류 (아/어/오/우/으/이/에)
- **Stage 2**: Stage 1에서 오 또는 우로 판정된 경우, Layer 5-7 평균 임베딩 → SVM → 오/우 이진 재판정

### 1.2 문제

Stage 2의 오/우 구분 정확도가 낮다.
특히 실제 녹음 환경에서 **우를 오로 오인하는 비율이 매우 높다.**

## 5. 실험 3: 실제 녹음 포함 LOSO

### 5.1 가설

실제 녹음 데이터를 학습에 포함하면, 같은 도메인의 다른 화자에 대해 일반화할 수 있을 것이다.

### 5.2 조건

- **데이터**: dataset 3화자(168개) + live 5화자(40개) = 총 208개
- **평가**: Leave-One-Speaker-Out (화자 8명, 돌아가며 1명 hold-out)

### 5.3 결과

| Hold-out 화자 | 전체 | 오 | 우 | 출처 |
|------|------|---|---|------|
| 20대남성 | 100% | - | 100% (5개) | live |
| 20대남성2 | 100% | - | 100% (5개) | live |
| 경상도여성 | 100% | 100% (5개) | 100% (5개) | live |
| 서울여성2 | 100% | 100% (5개) | 100% (5개) | live |
| Anna | 94.3% | 88% (25개) | 100% (28개) | dataset |
| 김동규 | 95.1% | 97% (35개) | 92% (26개) | dataset |
| 이은서 | 75.9% | 86% (28개) | 65% (26개) | dataset |
| **서울여성** | **50.0%** | **20% (5개)** | **80% (5개)** | **live** |

**전체 LOSO 결과:**

| | 전체 | 오 | 우 |
|---|---|---|---|
| 정확도 | **88.5%** | 88.3% | 88.6% |

혼동행렬:

|  | 예측:오 | 예측:우 |
|---|---|---|
| 정답:오 | 91 | 12 |
| 정답:우 | 12 | 93 |

### 5.4 해석

- 기존 모델(50%) 대비 **88.5%로 대폭 개선**
- 실제 녹음이 학습에 포함되면 다른 실제 녹음 화자도 잘 분류함
- **서울여성만 50%로 낮음**: 데이터가 10개뿐이고, 다른 화자들과 임베딩 분포가 다른 것으로 추정
- **LOSO의 한계**: 각 화자가 테스트될 때 자기 데이터는 학습에 안 들어가므로 개별 결과는 공정하나, live 데이터가 5~10개로 적어 통계적 신뢰도가 제한적

## 6. 임베딩 분포 시각화

**파일**: `ou_boundary_pca.png`

XLSR-53 Layer 5-7 임베딩을 PCA 2D로 투영 후, SVM 결정 경계와 함께 시각화.

### 6.1 관찰

1. **모음별 분포 (좌)**: 오와 우가 상당히 겹쳐 있으며, SVM 결정 경계가 복잡하게 구불구불함. 선형 분리 불가능.

2. **출처별 분포 (중)**: dataset(○)과 live(△)가 **서로 다른 영역에 분포**. dataset만으로 학습 시 live 영역의 경계를 학습하지 못하는 원인.

3. **화자별 분포 (우)**: 같은 화자의 오/우가 서로 가까이 클러스터링됨. **화자 간 변동(inter-speaker variability)이 오/우 간 차이(inter-vowel variability)보다 크다.**

### 6.2 근본 원인

Layer 5-7 임베딩은 음향적 특성뿐만 아니라 화자 정보도 강하게 인코딩한다. 이로 인해:
- 학습에 없는 새 화자의 임베딩이 기존 화자들과 다른 위치에 찍힘
- 해당 영역에서의 오/우 경계가 학습되지 않음
- 결과적으로 새 화자의 우를 오로 오인

## 7. 결론 및 향후 방향

### 7.1 결론

| 접근 | 결과 | 평가 |
|------|------|------|
| TTS 데이터 보강 | 효과 없음 (50% → 50%) | TTS-실제 녹음 도메인 갭 |
| 실제 녹음 LOSO | 대폭 개선 (50% → 88.5%) | 같은 도메인 데이터가 핵심 |

**핵심 발견**: 데이터의 양보다 **도메인 일치**가 중요하다. TTS 110개를 추가해도 효과가 없었지만, 실제 녹음 40개를 추가하자 38.5%p 개선되었다.

### 7.2 향후 시도 가능한 방향

1. **다른 레이어 탐색**: Layer 0 등 저층 레이어는 음향 특성을 더 잘 잡고 화자 정보가 적을 수 있음. `experiment_oh_oo_fix.py`에서 일부 탐색 완료.

2. **음향 특성 보조**: H1-H2 (1차/2차 하모닉 비율), F1/F2 포먼트 등 전통 음향 특성을 임베딩에 결합. `eval_stage2_improved.py`에서 시도.

3. **화자 적응 (Speaker Adaptation)**: 사용 초기에 사용자의 오/우 샘플 몇 개를 수집하여 분류기를 미세 조정.

4. **화자 정규화**: 임베딩에서 화자 정보를 제거하는 기법 (예: speaker adversarial training, CMVN 등).

5. **더 많은 실제 녹음 데이터 확보**: 다양한 화자의 실제 녹음을 추가하여 화자 독립적 경계 학습.
