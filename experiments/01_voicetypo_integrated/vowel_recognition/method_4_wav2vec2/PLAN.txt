method_4: wav2vec2 음소 인식
==============================

이것은 무엇인가:
  Facebook/Meta의 대규모 자기지도학습 음성 모델.
  수만 시간의 음성 데이터로 사전 학습되어 화자 독립적인 음소 인식 가능.
  캘리브레이션 없이도 동작할 수 있다는 게 최대 장점.

작동 원리:
  1. raw 오디오 (16kHz) 입력
  2. CNN 특성 추출기: 25ms 단위, 20ms 스트라이드로 잠재 표현 생성
  3. Transformer 인코더: 문맥을 고려한 고수준 표현
  4. CTC 디코더: 각 타임스텝에서 음소 예측
  5. 출력에서 모음 음소만 필터링

사용할 모델 후보:
  - vitouphy/wav2vec2-xls-r-300m-timit-phoneme (HuggingFace)
    -> IPA 음소 출력, 다국어, 300M 파라미터
  - 한국어 fine-tuned 모델 검색 필요

IPA 음소 -> 한국어 모음 매핑:
  /a/  -> 아
  /ʌ/  -> 어
  /o/  -> 오
  /u/  -> 우
  /ɯ/  -> 으
  /i/  -> 이
  /e/  -> 에
  /ɛ/  -> 애

장점:
  - 화자 독립적 (캘리브레이션 불필요)
  - 대규모 데이터로 학습되어 일반화 능력 높음
  - 노이즈에 강함

단점:
  - 모델 크기: ~1.2GB (메모리 부담)
  - 추론 지연: CPU에서 100-300ms (실시간성 떨어짐)
  - GPU 있으면 ~30ms까지 가능하지만 GPU 필수는 부담
  - 완전한 발화(문장) 단위로 설계됨 -> 지속 모음에 최적화 안 됨
  - 스트리밍 처리가 기본 지원 안 됨 (청크 단위 처리 필요)

최적화 가능성:
  - ONNX 변환으로 2-3배 속도 향상
  - 양자화(int8)로 모델 크기/속도 개선
  - 지속 모음만 분류하므로 짧은 청크(0.5초)로도 가능할 수 있음

필요 라이브러리:
  transformers       - HuggingFace 모델 로드 (pip install transformers)
  torch              - PyTorch 백엔드
  torchaudio         - 오디오 전처리
  sounddevice        - 마이크 입력

구현할 파일:
  model.py       - wav2vec2 모델 로드 + 추론
  classifier.py  - 음소 출력 -> 한국어 모음 매핑
  test.py        - 마이크 입력으로 실시간 테스트

예상 정확도: 95%+ (화자 무관)
지연: 100-300ms (CPU), ~30ms (GPU)
모델 크기: ~1.2GB
