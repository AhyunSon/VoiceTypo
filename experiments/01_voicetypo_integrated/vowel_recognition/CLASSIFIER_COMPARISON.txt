분류기 비교: 코사인 거리 vs SVM (RBF)
=========================================
상태: 미검증. 실제 테스트로 확인 필요.

======================================================================
A. 코사인 거리 (HTML 원본)
======================================================================

작동:
  - 각 모음의 프로토타입 (MFCC 벡터 1개씩) 저장
  - 입력 MFCC와 8개 프로토타입의 코사인 거리 계산
  - 가장 가까운 것 = 인식 결과

결정 경계: 선형 (보로노이)

장점:
  - 학습 불필요 (프로토타입만 있으면 됨)
  - 기본 프로토타입 제공 가능 -> 캘리브레이션 없이도 일단 동작
  - 계산 극도로 빠름 (<0.1ms)
  - 구현 10줄, 디버깅 쉬움

단점:
  - 모음당 대표점 1개 -> 발성 변동(퍼짐)을 모름
  - 선형 경계만 -> 분포가 겹치면 분리 불가
  - 신뢰도 계산이 임의적 (1위-2위 거리 차이)

======================================================================
B. SVM (RBF 커널)
======================================================================

작동:
  - 캘리브레이션 시 모음당 50~100개 MFCC 벡터 수집
  - scikit-learn SVC(kernel='rbf') 학습
  - 입력 MFCC -> predict() -> 모음 + predict_proba() -> 신뢰도

결정 경계: 비선형 (곡면)

장점:
  - 데이터 분포를 학습 -> 변동 범위 반영
  - 비선형 경계 -> 겹치는 모음(에/애, 으/어) 분리 가능성 높음
  - predict_proba()로 수학적 신뢰도

단점:
  - 캘리브레이션 필수 (기본 프로토타입으로 동작 불가)
  - 캘리브레이션 품질에 의존 (불성실하면 오히려 나빠질 수 있음)
  - scikit-learn 의존성 추가
  - 하이퍼파라미터(C, gamma) 기본값이 우리 데이터에 맞는지 모름

======================================================================
C. 아직 모르는 것들
======================================================================

  1. CMVN만 추가해도 코사인 거리가 충분히 좋아지는 것 아닌가?
     -> HTML 원본의 진짜 문제가 "분류기"인지 "정규화 부재"인지 분리 안 됨
     -> CMVN+코사인 vs CMVN+SVM 비교 필요

  2. 캘리브레이션 데이터 양이 SVM에 충분한가?
     -> 모음당 2초, ~50-80개 벡터, 8클래스 = 총 ~500개
     -> 13차원에서 500개: 일반적으로 충분하다고 하지만 확인 필요

  3. SVM 기본 하이퍼파라미터가 맞는가?
     -> 안 맞으면 캘리브레이션마다 GridSearch? -> 시간 증가

  4. 캘리브레이션 없이 쓸 수 있는 방법은?
     -> 코사인 거리는 기본 프로토타입으로 "일단 동작"이 가능
     -> SVM은 학습 데이터 없으면 아예 동작 불가
     -> 첫 실행 시 무조건 캘리브레이션 강제? 사용자 경험 문제

======================================================================
D. 검증 테스트 계획
======================================================================

동일 마이크 입력으로 4가지 조합 동시 비교:

  (a) raw MFCC + 코사인 거리     <- HTML 원본 그대로
  (b) CMVN MFCC + 코사인 거리    <- 정규화만 추가
  (c) raw MFCC + SVM             <- 분류기만 교체
  (d) CMVN MFCC + SVM            <- 둘 다 변경

이걸로 "정규화 효과"와 "분류기 효과"를 분리해서 측정 가능.
(b)가 (d)와 비슷하면 SVM 불필요 -> 코사인+CMVN이 답.
(c)가 (a)와 비슷하면 SVM만으로는 부족 -> CMVN이 핵심.
