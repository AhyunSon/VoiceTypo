VAD 비교: 수동 룰 기반 vs Silero VAD
========================================
상태: 미검증. 실제 테스트로 확인 필요.

======================================================================
A. 수동 룰 기반 VAD (HTML 원본)
======================================================================

작동:
  1. RMS 체크: rms > max(0.005, noiseFloor * 2.5)
  2. 피치 체크: 80Hz < frequency < 1100Hz (YIN으로 감지)
  3. 둘 다 만족 -> 음성
  4. 히스테리시스: ON 3프레임 연속, OFF 15프레임 연속

장점:
  - 추가 의존성 0 (numpy만)
  - 지연 0ms
  - 동작 원리 완전 투명 -> 디버깅 쉬움
  - "피치 있는 소리"만 감지 -> 모음 발성에 특화
  - torch 불필요 -> 설치 크기 작음

단점:
  - 임계값 하드코딩 -> 마이크/환경마다 동작 다를 수 있음
  - 노이즈 플로어 적응 느림 (EMA alpha=0.02)
  - 작은 소리 발성 시 놓칠 수 있음
  - 음악/TV도 피치 있으면 오탐 가능

======================================================================
B. Silero VAD
======================================================================

작동:
  - 사전 학습된 딥러닝 모델 (~1.5MB)
  - 30ms+ 오디오 입력 -> 음성 확률 (0.0~1.0) 출력
  - 확률 > threshold -> 음성

장점:
  - 검증된 정확도 (PyTorch Hub 공식, 기업 채택)
  - 다양한 환경/마이크/언어에서 학습 -> 강건
  - 무성음도 감지 가능
  - 비음성 소리(음악, 잡음) 구분 잘함
  - threshold 하나만 조절

단점:
  - torch 의존성 필수 (~2GB 설치 크기)
  - 모델 로드 ~1초 (앱 시작 시)
  - 16kHz 리샘플링 필요 (sounddevice 기본이 44100/48000Hz)
  - 블랙박스 -> 왜 이 구간을 음성으로 판단했는지 모름
  - "피치 있는 모음 발성"에 특화된 게 아님 -> 우리 용도에 과잉 사양?

======================================================================
C. 아직 모르는 것들
======================================================================

  1. 우리 시나리오에서 수동 VAD가 실제로 문제가 되는가?
     -> 사용자가 의도적으로 "아~" 하고 큰 소리로 발성
     -> 명확한 피치 있는 지속 모음
     -> 이런 케이스에서 수동 VAD도 잘 동작할 수 있음

  2. Silero의 강점이 우리한테 의미 있는가?
     -> 무성음 감지: 우리는 모음(유성음)만 필요 -> 의미 없음
     -> 비음성 구분: 사용 환경이 조용한 곳이면 의미 없음
     -> 노이즈 강건성: 시끄러운 환경이면 의미 있음

  3. torch 의존성 비용 대비 효과가 있는가?
     -> method_3만 테스트할 때 torch 2GB 설치가 부담
     -> method_4/5 (wav2vec2, whisper)도 쓸 거면 어차피 torch 필요
     -> 최종적으로 method_3만 채택하면 torch가 VAD 때문에만 필요한 상황

  4. 16kHz 리샘플링이 실시간 성능에 영향 주는가?
     -> 매 프레임 44100->16000 변환 오버헤드

======================================================================
D. 실용적 접근
======================================================================

  방안 1: 수동 VAD로 시작
    -> torch 없이 가볍게 시작
    -> 실사용 중 VAD 문제 발생하면 그때 Silero로 교체
    -> vad.py 인터페이스 통일해두면 교체 쉬움

  방안 2: 둘 다 구현해두고 설정으로 전환
    -> vad_manual.py + vad_silero.py
    -> 실행 시 --vad=manual / --vad=silero 선택

  방안 3: 수동 VAD 개선
    -> 임계값을 환경에 맞게 자동 조절 (적응형)
    -> Silero 수준은 아니지만 현재보다 나아질 수 있음

======================================================================
E. 검증 테스트 계획
======================================================================

동일 마이크 입력으로 두 VAD 동시 비교:

  테스트 시나리오:
  1. 큰 소리 모음 발성 (기본) -> 둘 다 잘 되는지
  2. 작은 소리 모음 발성 -> 수동 VAD가 놓치는지
  3. 무음 -> 둘 다 OFF인지
  4. 주변 소음 (음악, 대화) -> 오탐 비교
  5. 발성 시작/끝 -> 반응 속도 비교

  핵심 질문: "우리 실사용 환경에서 수동 VAD의 문제가 얼마나 큰가?"
  -> 크지 않으면 수동 VAD 유지 (의존성 최소화)
  -> 크면 Silero 도입 (정확도 우선)
