# Method 6: Embedding + SVM 실험 로그

> 최종 업데이트: 2026-03-09
> 목표: 8개 한국어 단모음(아,어,오,우,으,이,에,애) 화자 독립 실시간 인식

---

## 1. 실험 환경

- **데이터셋**: 525개 mp3 파일 (`vowel_recognition/dataset/`)
  - 화자 3명: Anna(TTS), 김동규(남), 이은서(여)
  - 조건: 클린(513), 리버브(3), 소음(9)
  - 모음별: 아(65), 어(70), 오(88), 우(80), 으(73), 이(73), 에(76), 애(0)
- **파이프라인**: 오디오 → wav2vec2 중간 레이어 → 에너지 상위 50% 프레임 → mean pooling → StandardScaler → SVM(RBF, C=10)
- **평가 방식**:
  - Stratified 5-Fold CV (같은 화자가 train/test에 섞임)
  - **LOSO** (Leave-One-Speaker-Out, 화자 독립 — 핵심 지표)
- **하드웨어**: Windows 11, CPU (GPU 없음)

### 10-3. 개선 실험 (experiment_oh_oo_fix.py)

진단 결과를 바탕으로 3가지 접근법을 순차적으로 시도.

#### 방법 2: F0(피치) + F1/F2(포먼트) 보조 특성 추가

**가설**: 오와 우의 F1 차이(오 561Hz vs 우 389Hz)를 명시적으로 알려주면 분류에 도움이 될 것.

오/우 음향 특성 비교:
```
         오              우           차이
F0_med:  249.0±78.9Hz    239.2±83.2Hz   +9.8   (거의 동일)
F1_med:  561.4±197.6Hz   389.0±291.4Hz  +172.4  (유의미한 차이)
F2_med:  3700.9±896.7Hz  3035.0±2025.7Hz +665.9  (차이 있으나 분산 큼)
F2-F1:   3139.5±803.2Hz  2646.0±1774.4Hz +493.5
```

**실험 결과 (오/우 이진 LOSO)**:

| 설정 | 전체 | 오 | 우 |
|------|------|---|---|
| 베이스라인 (임베딩만, 1024d) | 81.0% | 86.4% | 75.0% |
| 음향 특성만 (F0+F1/F2, 8d) | 72.0% | 60.2% | 85.0% |
| 임베딩 + 음향 (concat) | 81.0% | 85.2% | 76.2% |
| 임베딩 + 음향*5~50 (가중치) | 81.0% | 85.2% | 76.2% |
| 임베딩 + F1*20 | 81.5% | 86.4% | 76.2% |
| 임베딩 + (F0,F1,F2-F1)*20 | 81.5% | 86.4% | 76.2% |

**결론: 효과 거의 없음 (+0.5%)**
- F1 차이(172Hz)가 분명히 존재하지만, 1024d 임베딩이 이미 이 정보를 내포
- 가중치를 아무리 올려도 SVM이 RBF 커널에서 자동으로 스케일 조정 → 의미 없음
- 음향 특성만으로는 72.0%밖에 안 됨 → 분산이 너무 커서 단독으로도 부족

| L1-2 | 88.1% | 86.9% |
| L5-8 | 87.5% | 88.1% |

**핵심 발견**:
- **전체 7모음 최적 레이어(L16)와 오/우 최적 레이어(L20)가 다르다!**
- L16 오/우: 81.0% → L20 오/우: **88.1%** (+7.1%)
- 특히 우 정확도: 75% → **88%** (+13%)
- 낮은 레이어(L1, L5-7)도 오/우에서는 L16보다 우수
- 음향 특성 추가는 레이어 선택보다 효과가 미미

**해석**: L20은 XLSR-53의 상위 레이어로 더 추상적인 음소 표현을 학습. 전체 모음 분류에서는 L16이 범용적이지만, 오/우처럼 음향적으로 유사한 쌍에는 더 세분화된 표현이 유리.

#### 방법 1: 2단계 분류기 (전체→오/우 재판정)

**가설**: 1단계에서 L16으로 전체 7모음을 분류한 뒤, 오 또는 우로 판정된 샘플만 2단계에서 오/우 최적 레이어로 재판정하면 다른 모음 성능을 유지하면서 오/우만 개선 가능.

**실험 결과 (전체 7모음 LOSO)**:

| 설정 | 전체 7모음 | 오/우 | 오 | 우 |
|------|----------|------|---|---|
| 기존 (L16 단일) | 89.7% | 77.4% | 83.0% | 71.2% |
| L16→L16 재판정 | 89.9% | 78.0% | 84.1% | 71.2% |
| L16→L20 재판정 | **92.0%** | **84.5%** | 86.4% | **82.5%** |

| **L16→L5-7 재판정** | **92.2%** | **85.1%** | **86.4%** | **83.8%** |
| L16→L5-7+음향 재판정 | 92.2% | 85.1% | 86.4% | 83.8% |
| L16→L20-21+음향 재판정 | 92.0% | 84.5% | 85.2% | 83.8% |

### 10-4. 최종 결과 요약

#### 최적 전략: **L16 전체 분류 → L5-7 오/우 재판정 (2단계)**

| 지표 | 기존 (L16 단일) | **개선 (2단계)** | **변화** |
|------|---------------|----------------|---------|
| **전체 7모음 LOSO** | 89.7% | **92.2%** | **+2.5%** |
| 오 정확도 | 83.0% | **86.4%** | **+3.4%** |
| **우 정확도** | **71.2%** | **83.8%** | **+12.5%** |
| 오/우 통합 | 77.4% | **85.1%** | **+7.7%** |

- 다른 모음(아,어,으,이,에) 성능에 **악영향 없음**
- 추가 모델 로딩 불필요 (같은 XLSR-53 모델의 다른 레이어만 사용)

#### 시행착오에서 얻은 교훈

1. **음향 특성(F0/F1/F2) 추가는 효과 없음** — 사전학습 모델의 1024d 임베딩에 이미 내포. 가중치를 아무리 조절해도 개선 불가. 정보 자체가 부족한 것이 아니라 표현 공간에서 분리가 안 되는 문제.

2. **레이어 선택이 핵심** — 전체 모음 최적 레이어(L16)와 특정 혼동 쌍 최적 레이어(L20, L5-7)가 다르다. 과제별 최적 레이어가 다르다는 것은 각 레이어가 다른 수준의 음운 정보를 인코딩한다는 증거.

3. **2단계 접근이 정답** — 단일 레이어로 모든 모음을 구분하려는 것보다, 1단계에서 대분류 후 혼동 쌍만 전문 분류기로 재판정하는 것이 더 효과적. 다른 모음 성능을 건드리지 않고 문제 구간만 정밀 개선 가능.

4. **중앙 구간 잘라내기는 역효과** — 오디오의 시작/끝 전이구간이 오히려 분류에 유용한 정보를 포함. 안정 구간만 쓴다고 좋아지지 않음.

### 11-3. 음향 분석 — 왜 우가 오로 오인되는가?

#### 3화자 오/우 음향 특성 비교

| 화자 | 모음 | F0 (Hz) | F1 (Hz) | F2 (Hz) | H1-H2 (dB) |
|------|------|---------|---------|---------|-------------|
| 서울 여성 | 오 | 240 | 483 | 847 | -7.9 |
| 서울 여성 | 우 | 200 | 421 | 866 | -4.1 |
| 서울 여성 | **차이** | | **+62** | **-20** | **-3.8** |
| 경상도 여성 | 오 | 228 | 480 | 885 | -7.1 |
| 경상도 여성 | 우 | 227 | 426 | 892 | -2.6 |
| 경상도 여성 | **차이** | | **+55** | **-7** | **-4.5** |
| 20대 남성 | 우 | 158 | 403 | 1154 | +0.7 |

- **여성 2명**: F1 차이 55~62Hz, F2 차이 7~20Hz → 포먼트로 구별 거의 불가
- **H1-H2**: 여성은 오/우 간 3.8~4.5dB 차이 존재 (논문과 일치)
- **남성**: F0=158Hz로 여성(200~227Hz)과 확연히 구분, F2도 1154Hz로 다른 분포

#### 서울말 ㅗ/ㅜ 모음 융합 논문 참고

"서울말 /ㅗ/와 /ㅜ/를 구별하는 음향변수" (eksss.org) 등 연구에 따르면:
- **남성**: F1/F2로 오/우 잘 구별됨, H1-H2 차이 없음
- **여성**: F1/F2 공간에서 오/우가 거의 겹침, **H1-H2로 구별**
- 이 현상은 서울말에서 보고되었으나, 본 실험에서 **경상도 여성에서도 동일하게 나타남**
- → 20대 여성 전반의 현상일 가능성

### 11-5. Stage 2 개선 시도 — L0 + H1-H2

L0 임베딩과 H1-H2 음향 피처를 결합하여 Stage 2를 개선 시도.

#### 학습 데이터 LOSO 결과 (오/우 이진)

| 방법 | LOSO |
|------|------|
| L5-7 (현재) | **88.1%** |
| L0 | 67.3% |
| L0 + H1-H2 | 67.3% |

#### 실제 녹음 테스트 결과 (25개)

| 방법 | 오 전체 | 우 전체 | 합계 |
|------|---------|---------|------|
| L5-7 (현재) | 100% | 20% | 52% |
| L0 | 100% | 33% | 60% |
| L0 + H1-H2 | 100% | 33% | 60% |

- **L0는 남성 우를 100% 맞춤** (기존 40%→100%), 하지만 여성은 여전히 0%

- 학습 데이터 3명의 H1-H2 패턴이 새 화자에게 전이되지 않음

### 11-6. 근본 원인 진단

| **여성 모음 융합** | 여성 화자는 포먼트 공간에서 오/우가 근본적으로 겹침 | F2 차이 7~20Hz |

### 11-7. 현재 상태 및 향후 방향

**현재 상태**: 학습 데이터 내에서는 2단계 분류기가 효과적이나(우 83.8%), **새 화자에 대한 일반화가 실패**(우 20%). 이는 모델이나 피처의 문제라기보다 **학습 데이터 3명으로는 한국어 오/우의 화자 간 변동을 커버할 수 없는** 데이터 규모 문제.

**향후 방향**:
1. **학습 데이터 확보** — 다양한 성별/연령/방언 화자 추가 (최소 10~20명)
2. **Few-shot 화자 적응** — 새 화자가 오/우 각 3~5회 발화하여 SVM boundary 조정
3. **한국어 특화 사전학습 모델** — 한국어 fine-tuned wav2vec2 등 탐색
4. **문맥 활용** — 단모음이 아닌 음절 단위에서 초성/종성 정보 활용

### 11-8. 풀링 방식 비교 — energy_top50 vs middle_50

프레임 임베딩 풀링 전략을 변경하면 오/우 분리가 개선되는지 검증.

- **middle_50**: 시간축 중앙 50% 프레임만 평균 → 발화 안정 구간 활용

`main_integrated.py` — 음성 입력 → 모음 인식 → 타이포 변형 + 모핑 전체 통합 PySide6 앱.

```
┌─────────────────────────────────────────────────────────┐

          Main Thread            Background Thread
    ┌────────────┘               ┌──────┘
    │                            │
    │ IntegratedCanvas:          │ VowelRecognitionWorker:
    │ ├─ 피치→색상/스케일        │ ├─ XLSR-53 추론 (~360ms)

    │   (인식된 모음으로                (모음, 확신도)

    └───────────────────────────────────────────────────
```
