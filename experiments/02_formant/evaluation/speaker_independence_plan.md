# 화자 독립성 검증 계획

현재 baseline_simple.py 정확도 54.3% 는 **본인 1인 데이터** 기준.
미디어 아트 작품은 불특정 다수 관객을 대상으로 하므로,
화자 독립성 — 누가 발음해도 동작해야 함 — 이 핵심 요구사항.

본 문서는 이를 객관적으로 검증하기 위한 데이터 수집 + 측정 프로토콜.

---

## 1. 왜 본인 1인 데이터로는 부족한가

### 화자 간 변동성 (between-speaker variability)
- 성도 길이: 남성 ~17 cm, 여성 ~15 cm, 아동 ~12 cm → 같은 모음의 F2 가 200~600 Hz 범위로 변동
- 방언/지역 영향: 영남·호남·서울 화자의 모음 공간 미묘한 차이
- 연령: 노년층은 모음 공간 축소 (vowel space reduction)
- 개인 발음 습관: 입 벌림 정도, 입술 둥글기 변동

→ 1인 데이터 정확도 X% 는 다른 화자에서 X-30%~X+10% 로 들쭉날쭉 나올 가능성 큼.

### 위험 시나리오
본인 데이터에 _REFS 를 맞추면 (방금 사용자가 명시적으로 금지) 본인 정확도 90% 도 가능. 그러나
- 다른 여성 화자: 50% (음역 다름)
- 남성 화자: 30% (성도 길이 다름)
- 아동: 20% (성도 짧음, F0 매우 높음)

이러면 작품으로 동작 불가능. **다른 화자 데이터 없이는 진짜 정확도를 알 수 없음.**

---

## 2. 필요 화자 규모 (현실 vs 이상)

### 통계적 이상 (참고)
- 학계 음성학 논문 평균: 30~50명 화자 / 성별
- 머신러닝 ASR: 수천 시간 / 수천 명

### 현실적 최소 (인턴 환경 가정)
**1차 검증 (필수)**: 6 ~ 10명
- 여성 3명 + 남성 3명 (성별 차이 확인)
- 본인은 별도 (학습/튜닝 영향 격리)

**2차 보강 (가능 시)**: 추가 10명
- 다양한 연령 (20대 / 30~40대 / 50대+)
- 가능하면 1명 정도 아동 (작품에서 어린 관객 대상이라면)

**3차 (작품 발표 직전 검증)**: 5 ~ 10명
- 작품 발표 환경에서 즉석 녹음 (관객 후보 시뮬레이션)
- 이때는 이미 시스템이 안정된 상태에서 사후 검증용

**최소 6명 → 권장 16명 → 안전 26명**.

### 인턴 환경 제약 고려한 모집안
- **1차 (즉시 가능)**: 같은 인턴실/스튜디오/연구실 동료 — 5~10명. 1주일 내 완료.
- **2차 (시간 필요)**: 친구/가족 — 추가 5~10명. 2~3주.
- **3차**: 작품 설치 단계 — 관람객/방문자에게 30초 협조 요청.

## 3. 모집 시 데모그래피 균형

| 카테고리 | 권장 인원 | 이유 |
|---|---:|---|
| **여성 / 20대** | 3 | 본인과 가까운 카테고리, 차이 작은지 확인 |
| **여성 / 30~50대** | 3 | 다른 음역대 |
| **남성 / 20대** | 3 | 성별 차이 핵심 검증 |
| **남성 / 30~50대** | 3 | 음역대 추가 |
| **여성 / 60대+** | 1~2 | 노년 음성 (가능 시) |
| **아동 / 7~12세** | 1~2 | 고음역 + 성도 짧음 (작품 대상에 따라) |

**최소 12 명 (위 표 합계)** 권장.

방언이나 지역은 1차 검증에선 균형 안 맞춰도 됨 (서울 표준어 위주). 2차 이상에서 확인.

## 4. 측정 프로토콜

기존 [evaluation/record_dataset.py](evaluation/record_dataset.py) 와 동일하게.

### 4.1 1인당 녹음 분량
- 한국어 단모음 7개 × 5회 = 35 wav (1인당)
- 1.5초/회 → 1인당 ~1분 + 카운트다운 시간 합쳐 ~5분

### 4.2 환경 통제
- 같은 마이크 (가능하면 본인 환경의 마이크)
- 같은 거리 (~30cm)
- 조용한 실내 (RMS noise 0.005 이하)
- 단독 모음 발화 (자음 없이)
- 1.5초 동안 일정한 음량 유지

### 4.3 파일명 규칙 (확장)
현재: `{모음}_{회차:02d}.wav`
다중 화자: `evaluation/dataset_speakers/{speaker_id}/{모음}_{회차:02d}.wav`

`speaker_id` = `{성별}{연령대}_{이름이니셜}` (예: `f20_KH`, `m30_JS`).
이름은 **익명화** — 데이터 외부 공개 가능성 고려.

### 4.4 메타데이터
각 화자 폴더에 `meta.json`:
```
{
  "speaker_id": "f20_KH",
  "gender": "female",
  "age_range": "20-29",
  "dialect": "seoul",
  "consent": true,
  "recorded": "2026-04-XX"
}
```

**개인정보 보호**: 이름·연락처 등 직접 식별 가능 정보 저장 금지. ID 와 broad 카테고리만.

## 5. 측정 항목

각 화자 데이터에 대해 [evaluation/baseline_simple.py](evaluation/baseline_simple.py) 변형판(다중 화자 지원)을 적용:

### 5.1 기본 지표
- 화자별 전체 정확도 (Y/35)
- 화자별 모음별 정확도 (Y/5 × 7)
- 화자별 혼동 행렬

### 5.2 종합 지표
- **Mean accuracy across speakers**: 모든 화자 평균
- **Worst-speaker accuracy**: 가장 낮은 화자 (작품 강건성의 진짜 척도)
- **Std across speakers**: 화자 간 변동
- **Per-vowel cross-speaker accuracy**: 각 모음이 모든 화자에서 잘 인식되는지

### 5.3 분석 지표 (실패 원인 진단용)
- 화자별 raw F1/F2 분포 → _REFS ellipse 와 비교
- 어느 화자가 _REFS 와 가장 멀리 떨어지는가
- 그 화자의 F0 (음역대) 와 정확도 상관관계

## 6. 합격 기준 (proposed)

작품 배포 가능 여부 결정 기준:

| 지표 | 합격선 | 우수 |
|---|---:|---:|
| 평균 정확도 (전 화자) | ≥ 60% | ≥ 75% |
| 최악 화자 정확도 | ≥ 45% | ≥ 60% |
| 화자 간 std | ≤ 15%p | ≤ 8%p |
| 모음별 최저 (전 화자 평균) | ≥ 40% | ≥ 60% |

**합격선 미달 시**: 코드 추가 단순화·_REFS 보강·시각화 단순화(애매한 모음 거부) 중 선택.
**합격선 만족 시**: 작품 배포 진행.

## 9. 리스크 및 대비

| 리스크 | 대비 |
|---|---|
| 화자 모집 어려움 (시간 부족) | 1차 6명만으로도 분산은 측정 가능. 그 후 점진 확대. |
| 마이크/환경 차이로 측정 일관성 저하 | 같은 마이크/공간 통일. 변경 시 별도 기록. |
| 어떤 화자에서 0% 가까운 결과 | 이건 발견 자체가 가치. _REFS 또는 시스템 한계 노출. |
| 작품 발표 환경의 잡음 | Phase 5 발표 환경 녹음에서 별도 검증. 필요 시 VAD 파라미터 조정. |
| 화자가 익숙하지 않은 발음 (외국인 한국어 등) | 1차 검증에서는 제외. 작품 시나리오에 따라 별도 확장 검토. |

## 10. 본 계획의 한계

- 12 ~ 20명 표본은 학계 기준 작은 규모. 통계 결론은 "시사적" 수준.
- 자연 발화(문장) 가 아닌 단독 모음 측정 → 실제 작품 시나리오와 일치하지만 학계 비교는 어려움.
- 마이크/방 차이는 화자 간 측정에서 중요 변수 — 가능한 한 통일하지만 완벽 통제 불가.

이 한계는 본 작품의 목적 (예술 작품, 학술 논문 아님) 을 고려하면 수용 가능.
